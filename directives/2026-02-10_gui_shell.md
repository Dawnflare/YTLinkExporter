# Directive: GUI Shell

## 1. Objective
Build the CustomTkinter GUI: root window, header (URL input), filter panel, export checkboxes, progress/status log, and background threading.

## 2. Context & Research (Context-First)
- PRD §4 specifies CustomTkinter, dark/light theme, and a detailed layout.
- All backend logic (core + exporters) is now available.

## 3. Planning & Risk Assessment
- **Risk Level:** MEDIUM — multi-file GUI assembly with threading.

## 4. Execution Steps (Scripts)
1. Create `src/utils/threading.py`.
2. Create `src/gui/status.py` (progress + log).
3. Create `src/gui/header.py` (URL input + Load Metadata).
4. Create `src/gui/filters.py` (collapsible filter panel).
5. Create `src/gui/export_opts.py` (checkboxes + path).
6. Create `src/gui/app.py` (root window, assembly, Export pipeline).

## 5. Validation Standard
- `python src/main.py` launches the window without errors.
- All widgets render and respond to clicks.

## 6. Expected Deliverables
- Six new modules in `src/gui/` and `src/utils/`.

## 7. Failure Handling
- GUI errors surface in the status log via try/except wrappers.
